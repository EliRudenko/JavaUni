# Питання 1. Загальна характеристика мови програмування Python

## Коротка відповідь
Python — інтерпретована високорівнева мова 3-го покоління з динамічною сильною типізацією, яка підтримує кілька парадигм (процедурну, ООП, функціональну). Вона зручна для швидкої розробки, має багату стандартну бібліотеку, але поступається за швидкодією компільованим мовам.

## Детально (простою мовою)
- **Покоління мови.** Python відносять до мов 3-го покоління (3GL): програміст працює з абстракціями, а не з машинними інструкціями. Це видно по звичайному “людському” синтаксису у файлах проєкту.
- **Парадигми.** Python дозволяє:
  - писати процедурний код (функції та виклики),
  - будувати ООП (класи і об’єкти),
  - використовувати функціональний стиль (lambda, генератори, спискові включення).
- **Тип виконання.** Python — **інтерпретована** мова (байт-код виконується інтерпретатором). У CGI-скриптах це видно по shebang, який показує шлях до інтерпретатора, і по виконанню скрипта без компіляції.
- **Типізація.** Типізація **динамічна** (типи визначаються під час виконання) та **сильна** (не всі типи можна неявно змішувати). Це видно у прикладах перетворень типів у навчальних файлах.
- **Переваги.** Швидкий старт, читабельність, багато бібліотек, велика спільнота.
- **Недоліки.** Нижча швидкодія порівняно з C/C++/Java, потрібні оптимізації для великих навантажень, інтерпретаторна природа.

## Де в коді (приклади)
- **Інтерпретація і CGI-виконання:** у CGI-скриптах є shebang, що вказує інтерпретатор (`cgi/index.py`, `cgi/access_manager.py`).
- **Багатопарадигменність:**
  - процедурно: `basics/3_funcs.py`,
  - ООП: `basics/8_oop1.py`, `basics/9_oop2.py`,
  - функціонально: `basics/4_lambda.py`, `basics/12_generators.py`.
- **Динамічна типізація:** приклади з різними типами у `basics/2_types.py`.

## Ключевые моменты в коде
- `basics/2_types.py` — показаны базовые типы и динамическая типизация (тип определяется значением переменной).
- `basics/3_funcs.py` — демонстрация функций и процедурного стиля как одной из парадигм Python.
- `basics/8_oop1.py`/`basics/9_oop2.py` — ООП-парадигма: классы, методы, инкапсуляция.
- `basics/4_lambda.py`/`basics/12_generators.py` — функциональный стиль: lambda и генераторы.
- `cgi/index.py`/`cgi/access_manager.py` — CGI-скрипты показывают интерпретируемое выполнение через shebang.

## Куди перейти в коді
- Перегляньте **`cgi/index.py`** і **`cgi/access_manager.py`** — там видно shebang та принцип інтерпретованого запуску у CGI.
- Перегляньте **`basics/2_types.py`**, **`basics/3_funcs.py`**, **`basics/8_oop1.py`**, **`basics/9_oop2.py`**, **`basics/4_lambda.py`**, **`basics/12_generators.py`** — там демонструються різні парадигми та типи.
